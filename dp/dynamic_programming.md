# 动态规划（Dynamic Programming）核心概念精讲

## 1. 什么是动态规划？

**动态规划**是一种通过将原问题分解为**相互重叠的子问题**，并**存储子问题的解**来避免重复计算，从而高效求解最优化问题的算法思想。

> 核心口诀：**“记住求过的解，避免重复计算”**

### 与分治法的区别

- **分治法**（如归并排序）将问题分解成**互不相交**的子问题，分别求解再合并。
- **动态规划**适用于子问题**大量重叠**的情况，如果还用分治，会反复求解相同子问题，效率极低。

------

## 2. 动态规划的核心概念

要掌握 DP，你必须理解以下四个关键词：

### 2.1 最优子结构（Optimal Substructure）

**如果一个问题的最优解包含其子问题的最优解**，则称该问题具有最优子结构性质。
*简单说：大问题的最优解可以由小问题的最优解推导出来。*

例如：在最短路径问题中，如果 A→C 的最短路径经过 B，那么 A→B 的路径也一定是 A→B 的最短路径。

### 2.2 重叠子问题（Overlapping Subproblems）

在求解过程中，**相同的子问题会被反复计算多次**。
例如：斐波那契数列 `F(n) = F(n-1) + F(n-2)`，计算 F(5) 时需要 F(4) 和 F(3)，而 F(4) 又需要 F(3) 和 F(2)，这里 F(3) 被重复计算了两次。

### 2.3 状态（State）

**状态是描述问题当前所处情形的一个“快照”**。通常用一组变量（如数组下标、剩余容量等）来定义。
例如：`dp[i]` 表示“前 i 个物品能得到的最大价值”，这里的 `i` 就是一个状态变量。

### 2.4 状态转移方程（Transition Equation）

**状态之间如何递推的公式**，它是 DP 的核心。
例如：对于斐波那契数列，状态转移方程就是 `dp[i] = dp[i-1] + dp[i-2]`。

------

## 3. 动态规划的两种实现方式

| 方式         | 思路                                                         | 优缺点                     |
| :----------- | :----------------------------------------------------------- | :------------------------- |
| **自顶向下** | 从原问题出发，递归求解子问题，并用记忆化数组记录已算过的结果。 | 直观，但递归有栈溢出风险。 |
| **自底向上** | 从最小子问题开始，按顺序递推，直到原问题。                   | 迭代实现，效率高，常用。   |

两种方法本质相同，只是计算顺序相反。初学者可以先掌握自底向上（表格法）。

------

## 4. 动态规划解题五步法

当你面对一道可以用 DP 解决的问题时，按以下步骤思考：

1. **定义状态**：如何用数组（或高维数组）描述问题的子问题？
   → 问自己：`dp[i]`、`dp[i][j]` 分别代表什么？
2. **确定状态转移方程**：当前状态如何从前面的状态推导出来？
   → 问自己：要计算 `dp[i]`，需要知道哪些更小的状态？
3. **初始化**：最小的子问题的值是什么？
   → 通常需要设置 `dp[0]`、`dp[0][0]` 等边界条件。
4. **确定计算顺序**：一般从规模小的状态向规模大的状态计算。
   → 确保在计算 `dp[i]` 时，它所依赖的子状态都已经算好了。
5. **返回最终结果**：通常返回 `dp[n]` 或 `dp[m][n]`。

## 5. 经典问题详解（从易到难）

下面通过几个经典问题，让你感受 DP 的思考过程。

### 5.1 斐波那契数列（入门）

**问题**：求第 n 个斐波那契数（F(0)=0, F(1)=1, F(n)=F(n-1)+F(n-2)）。

**分析**：

- 最优子结构：F(n) 由 F(n-1) 和 F(n-2) 组成，显然子问题最优就是本身。
- 重叠子问题：递归树中有大量重复计算。
- 状态：`dp[i]` 表示第 i 个斐波那契数。
- 转移方程：`dp[i] = dp[i-1] + dp[i-2]`
- 初始化：`dp[0]=0, dp[1]=1`
- 计算顺序：从 i=2 到 n

**代码（自底向上）**：


```cpp
#include <vector>
using namespace std;

// 求第 n 个斐波那契数（F(0)=0, F(1)=1）
int fib(int n) {
    if (n < 2) return n;
    vector<int> dp(n + 1);
    dp[0] = 0;
    dp[1] = 1;
    for (int i = 2; i <= n; ++i) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    return dp[n];
}
```

**空间优化**：其实只需要两个变量滚动更新，但初学阶段先写数组版有助于理解。

### 5.2 爬楼梯问题（一维DP）

**问题**：每次可以爬 1 或 2 个台阶，爬到第 n 阶有多少种不同方法？

**思考**：

- 到达第 n 阶的最后一步，可能是从 n-1 阶跨 1 步，或从 n-2 阶跨 2 步。
- 所以 `dp[n] = dp[n-1] + dp[n-2]`，和斐波那契一样！
- 初始化：`dp[1]=1, dp[2]=2`（或者 dp[0]=1 作为边界，看个人定义）

**代码**：


```cpp
#include <vector>
using namespace std;

// 爬到第 n 阶楼梯的方法数（每次爬 1 或 2 阶）
int climbStairs(int n) {
    if (n <= 2) return n;
    vector<int> dp(n + 1);
    dp[1] = 1;
    dp[2] = 2;
    for (int i = 3; i <= n; ++i) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    return dp[n];
}
```



### 5.3 最长公共子序列（LCS）（二维DP）

**问题**：给定两个字符串 text1 和 text2，返回它们的最长公共子序列的长度。

**分析**：

- 定义状态：`dp[i][j]` 表示 text1 的前 i 个字符 和 text2 的前 j 个字符 的最长公共子序列长度。
- 转移方程：
  - 如果 `text1[i-1] == text2[j-1]`（注意索引从0开始），则当前字符可以加入子序列：`dp[i][j] = dp[i-1][j-1] + 1`
  - 否则，取 `max(dp[i-1][j], dp[i][j-1])`（分别表示舍弃 text1 的当前字符或 text2 的当前字符）
- 初始化：`dp[0][j] = 0`，`dp[i][0] = 0`（空串与任何串的LCS为0）
- 计算顺序：i 从1到m，j 从1到n 逐行或逐列均可。
- 结果：`dp[m][n]`

**代码**：


```cpp
#include <vector>
#include <string>
#include <algorithm>
using namespace std;

int longestCommonSubsequence(string text1, string text2) {
    int m = text1.size(), n = text2.size();
    vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));
    for (int i = 1; i <= m; ++i) {
        for (int j = 1; j <= n; ++j) {
            if (text1[i - 1] == text2[j - 1]) {
                dp[i][j] = dp[i - 1][j - 1] + 1;
            } else {
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
            }
        }
    }
    return dp[m][n];
}
```



### 5.4 0-1 背包问题（二维DP优化为一维）

**问题**：有 N 件物品，每件物品重量为 w[i]，价值为 v[i]，背包容量为 W。每件物品只能选或不选，求能装入的最大总价值。

**分析**：

- 状态：`dp[i][j]` 表示考虑前 i 件物品，背包容量为 j 时能获得的最大价值。
- 转移方程：
  - 不选第 i 件：`dp[i][j] = dp[i-1][j]`
  - 选第 i 件（前提 j >= w[i]）：`dp[i][j] = dp[i-1][j - w[i]] + v[i]`
  - 取两者最大值
- 初始化：`dp[0][j] = 0`（没有物品时价值为0）
- 计算顺序：i 从1到N，j 从0到W。
- 结果：`dp[N][W]`

**代码（二维）**：

```cpp
#include <vector>
#include <algorithm>
using namespace std;

// N: 物品数量, W: 背包容量, w: 重量数组, v: 价值数组
int knapsack_2d(int N, int W, vector<int>& w, vector<int>& v) {
    vector<vector<int>> dp(N + 1, vector<int>(W + 1, 0));
    for (int i = 1; i <= N; ++i) {
        for (int j = 1; j <= W; ++j) {
            if (j >= w[i - 1]) {
                dp[i][j] = max(dp[i - 1][j], dp[i - 1][j - w[i - 1]] + v[i - 1]);
            } else {
                dp[i][j] = dp[i - 1][j];
            }
        }
    }
    return dp[N][W];
}
```



**空间优化（一维滚动数组）**：

- 观察发现，`dp[i][j]` 只依赖于 `dp[i-1][...]`，因此可以用一维数组 `dp[j]` 表示当前层。
- 但要注意 **内层循环必须从大到小遍历**，以保证 `dp[j-w[i]]` 是上一层的值（如果从小到大，会被当前层覆盖）。

```cpp
#include <vector>
#include <algorithm>
using namespace std;

int knapsack_1d(int N, int W, vector<int>& w, vector<int>& v) {
    vector<int> dp(W + 1, 0);
    for (int i = 0; i < N; ++i) {
        // 必须从大到小遍历容量，保证每个物品只选一次
        for (int j = W; j >= w[i]; --j) {
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
        }
    }
    return dp[W];
}
```

## 6. 常见DP类型与识别技巧

| 类型           | 特点                                                         | 常见问题                          |
| :------------- | :----------------------------------------------------------- | :-------------------------------- |
| **线性DP**     | 状态在一维数组上递推，如斐波那契、最大子数组和。             | 爬楼梯、打家劫舍                  |
| **区间DP**     | 状态定义为 `dp[i][j]` 表示区间 [i,j] 上的最优解，通常需要枚举分割点。 | 石子合并、最长回文子序列          |
| **树形DP**     | 在树上进行动态规划，通常用 DFS 后序遍历。                    | 二叉树最大路径和、树的最大独立集  |
| **状态压缩DP** | 当状态维度很高时，用二进制位表示状态，常用于集合覆盖问题。   | 旅行商问题（TSP）、最短哈密顿路径 |

**如何识别一道题适合用DP？**

- 问题要求 **最优解**（最大值、最小值、最少方式等）。
- 问题可以分解为相似的子问题，且子问题有重叠。
- 无后效性：**一旦某个阶段的状态确定，之后的过程不受之前状态的影响**（即未来与过去无关，只与当前状态有关）。

------

## 7. 优化技巧简介

- **空间优化（滚动数组）**：如上背包例子，将二维降为一维。
- **四边形不等式优化**：用于某些区间DP，可将 O(n³) 降为 O(n²)，但理解门槛较高。
- **斜率优化 / 凸包优化**：用于特定形式的 DP 转移方程。

初学者先掌握基础 DP 和空间优化即可。

## 8. 练习建议与资源

1. **循序渐进**：从线性DP（如 LeetCode 70、198、300）开始，再到二维DP（1143、72），最后尝试背包、区间DP。
2. **画表理解**：遇到二维DP，自己在纸上画个表格，手动模拟填表过程，理解每个格子如何得来。
3. **总结套路**：比如“最长公共子序列”是经典的“相等时加1，不等时取max”模式。
4. **常用网站**：LeetCode 题库的“动态规划”标签、LintCode、Codeforces 的 DP 题单。

------

动态规划之所以叫“动态”，是因为状态在“规划”中转移。一开始可能会觉得抽象，但多做几道经典题，慢慢你会形成条件反射：看到问题就思考“能否定义状态？转移方程是什么？”。

**记住：DP 不是魔法，而是一种系统性的思维框架。**
