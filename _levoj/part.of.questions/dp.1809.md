# P1809 wzy的跑步

#### 题目描述

wzy 在跑步，由于昨天下雨，跑道上某些位置有积水。wzy 不想踩到水坑，踩到前会选择跳过去。具体来说：

- 跑道抽象为长度为 `n` 的序列，wzy 从位置 `1` 出发，目标到达位置 `n`。
- 跑道上有 `m` 个位置存在积水。
- 每次可以跳 `1` 到 `k` 单位长度的正整数距离。

问 wzy 最少踩到几次水坑能到达位置 `n`。

#### 输入描述

输入第一行三个正整数 `n`，`m`，`k`，分别代表：

- 路径的长度
- 积水的个数
- wzy 一次最远可以跨的距离

第二行共 `m` 个正整数 `a_i`，分别代表第 `i` 个水坑的位置。

#### 输出描述

输出一行一个正整数，表示 wzy 至少踩到几次水坑才能到达位置 `n`。

#### 样例输入

```
6 3 2
2 3 5
```

#### 样例输出

```
1
```

#### 样例解释

位置 `2`、`3`、`5` 存在水坑。wzy 从位置 `1` 出发，可行走法为 `1→2→4→6`，仅踩到 `1` 次水坑，这是最少次数的走法之一。

#### 数据规模

- `n ≤ 10000`
- `m ≤ n`
- `k ≤ 50`

## 题目解析：P1809 wzy的跑步

### 问题理解

- 跑道长度为 `n`，从位置 1 出发，要到位置 `n`。
- 有 `m` 个位置有积水（不能踩，但可以选择跳过去，跳不过去就只能踩）。
- 每次可以跳 `1` 到 `k` 单位长度（正整数步长）。
- 目标：**最少踩到几次水坑**到达终点。

### 思路分析

这是一个典型的**动态规划**最优化问题。
因为步长最大为 `k`，所以到达位置 `i` 只能从 `[i-k, i-1]` 这些位置跳过来。
我们只需要记录到达每个位置时的最少踩水次数，然后逐步递推。

### 为什么是 dp?

这涉及两个关键点：**最优子结构**和**无后效性**。

#### 1. 最优子结构

问题要求从位置 1 到位置 n 的最少踩水次数。考虑到达位置 i 的最后一步：它一定是从某个位置 j（满足 i−k≤j<i*i*−*k*≤*j*<*i*）跳过来的。如果最后一步是从 j 跳到 i，那么到达 i 的总踩水次数 = 到达 j 的最少踩水次数 + (i 是否为水坑 ? 1 : 0)。

现在假设存在一条到达 i 的最优路径，记其最后一步来自 j，则这条路径的前半段（到达 j）也必然是从 1 到 j 的最优路径。否则，如果存在一条到达 j 的更优路径，那么用这条更优路径替换前半段，就能得到一条到达 i 的更优路径，与假设矛盾。因此，问题具有最优子结构：全局最优解包含局部子问题的最优解。

#### 2. 无后效性

当前决策（选择从哪个 j 跳过来）只依赖于之前的状态（dp[j]），而不会受到未来状态的影响。因为步长限制为 k，我们只需要知道到达前面位置的最少踩水次数，而不需要知道具体路径，未来只关心当前结果。这满足无后效性，可以动态规划。

#### 3. 递推关系覆盖所有可能

由于每次跳跃距离在 1 到 k 之间，任何到达 i 的路径最后一步必然落在区间 [i−k,i−1][*i*−*k*,*i*−1] 内。因此，我们枚举所有可能的 j，取最小值，就能覆盖所有可能的最后一步，从而找到全局最优。
#### 状态定义

令 `dp[i]` 表示从起点 **1** 到达位置 **i** 时，**最少踩到水坑的次数**。

#### 状态转移

如果当前位置 `i` 本身是水坑，那么从某个前驱位置 `j` 跳到 `i` 时，会额外增加一次踩水（即 `+1`）；否则 `+0`。

转移方程：

```
dp[i] = min( dp[j] + (water[i] ? 1 : 0) )   对于所有 j ∈ [max(1, i-k), i-1]
```

其中 `water[i]` 为 `true` 表示 `i` 是水坑。

#### 边界条件

- 起点 `dp[1] = (water[1] ? 1 : 0)`，因为一开始就在 1 号位置，如果 1 是水坑，就算踩到一次。
- 其他位置初始化为一个很大的数（如 `INF`）。

#### 最终答案

`dp[n]` 即为所求。

#### 复杂度

- 时间复杂度：`O(n * k)`，最坏 `10000 * 50 = 5e5`，可以接受。
- 空间复杂度：`O(n)`，可以用数组存储。

### 代码实现（C++）

```
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    int n, m, k;
    cin >> n >> m >> k;
    vector<bool> water(n + 1, false);   // 标记水坑
    for (int i = 0; i < m; ++i) {
        int pos;
        cin >> pos;
        water[pos] = true;
    }

    const int INF = 1e9;
    vector<int> dp(n + 1, INF);
    dp[1] = water[1] ? 1 : 0;            // 起点初始化

    for (int i = 2; i <= n; ++i) {
        int add = water[i] ? 1 : 0;      // 如果当前位置是水坑，踩到要加1
        for (int j = max(1, i - k); j < i; ++j) {
            dp[i] = min(dp[i], dp[j] + add);
        }
    }

    cout << dp[n] << endl;
    return 0;
}
```

### 样例验证

输入：


```
6 3 2
2 3 5
```


计算过程：

- 水坑：2,3,5
- dp[1] = 0
- i=2: add=1，j∈[1,1] → dp[2] = dp[1]+1 = 1
- i=3: add=1，j∈[1,2] → min(dp[1]+1=1, dp[2]+1=2) = 1
- i=4: add=0，j∈[2,3] → min(dp[2]+0=1, dp[3]+0=1) = 1
- i=5: add=1，j∈[3,4] → min(dp[3]+1=2, dp[4]+1=2) = 2
- i=6: add=0，j∈[4,5] → min(dp[4]+0=1, dp[5]+0=2) = 1
  输出 `1`，符合样例。

### 注意事项

- 如果起点或终点本身就是水坑，也会计入踩水次数，这与题意一致。
- 当 `k` 较大时，注意内层循环的起始位置不要小于 1。
- 若无法到达终点（但题目保证 m ≤ n 且总能到达？这里没有明确，但按照题意总可以一步一步跳，所以一定能到），dp[n] 不会是 INF。
